# 竞赛/比赛策略参考指南

> 本文档是 resume-builder skill 的参考文件，用于在 Phase 1（方向确认）和 Phase 2（技能盘点）阶段为用户提供竞赛相关的策略建议。竞赛是简历中仅次于实习的第二加分项，合理规划竞赛经历可以显著提升简历竞争力。

---

## 一、竞赛在简历中的价值

### 1.1 为什么竞赛重要？

竞赛经历在简历筛选阶段具有多重价值：

1. **能力证明信号**：竞赛获奖是对技术能力的第三方认证，比自述"精通XXX"更有说服力。HR 在 6-10 秒的简历扫视中，竞赛获奖是明确的"信号点"。

2. **仅次于实习的加分项**：简历模块权重排序为：实习经历 > 竞赛获奖 > 项目经历 > 论文/专利 > 社团/学生工作。没有实习经历时，竞赛是最有效的替代加分手段。

3. **弥补学历劣势**：对于双非/二本/专科同学，竞赛是突围关键。一份有 Kaggle 银牌或天池 Top 10% 的简历，可让 HR 在学历筛选时多停留 3-5 秒。

4. **大厂关键词命中**：大厂简历系统自动筛选时，竞赛名称（"ACM""Kaggle""天池""蓝桥杯"）是高频命中词。

5. **面试话题素材**：竞赛经历是极佳的面试素材——"你在竞赛中做了什么？""遇到什么困难？"这些问题有充分的准备空间。

### 1.2 不同类型竞赛的简历权重

| 级别 | 示例 | 简历价值 | 说明 |
|------|------|---------|------|
| 国际级 | Kaggle 金牌/银牌、ACM-ICPC World Finals | ★★★★★ | 含金量极高，直接成为简历亮点 |
| 国际级（参与） | Kaggle 铜牌/Top 10%、 Google Summer of Code | ★★★★ | 即使未获顶级名次，国际赛事参与本身就说明能力 |
| 国家级（获奖） | 天池 Top 10、蓝桥杯国一/国二、CCPC 金奖 | ★★★★ | 国内大厂认可度高 |
| 国家级（参与） | 天池参赛、蓝桥杯省赛 | ★★★ | 有比没有好，可转化为项目经历 |
| 省级 | 蓝桥杯省二/省三、省赛奖项 | ★★☆ | 对中小厂有用，大厂边际效益递减 |
| 校级 | 校内编程比赛、课程设计比赛 | ★ | 不建议写入简历，除非完全没有其他经历 |

> **重要原则**：校级竞赛不建议写入简历。简历上写校级竞赛反而会让 HR 觉得你的能力天花板较低——"这个人最好的竞赛只是校级的？"

### 1.3 竞赛 vs 项目 vs 实习的互补关系

```
高竞争力简历的理想组合:
  实习经历（1-2段） + 竞赛获奖（1-2个） + 项目经历（2-3个）

替代方案（无实习）:
  竞赛获奖（1-2个，至少一个是高级别） + 项目经历（3-4个，包含竞赛转化的项目）
```

## 三、竞赛策略（按学历分层）

### 3.1 985/211 学生

**定位**：竞赛是锦上添花，不是雪中送炭。

**策略**：
- 只冲高级别竞赛。简历上写一个 Kaggle 银牌比写三个省级二等奖有效 10 倍。
- 1-2 个高质量竞赛经历足够，宁缺毋滥。
- 如果已经有多段实习经历，竞赛的重要性降低，可以把时间投入实习。

**重要提醒**：
> 没有竞赛经历也比写"水竞赛"好。简历上出现低质量竞赛反而会拉低面试官对你的预期——"985 的学生就这水平？"

**推荐选择**：
- 算法方向：Kaggle 银牌+ 或 天池 Top 5%
- 后端方向：ACM/ICPC 区域赛以上 或 华为软件精英挑战赛 Top 50
- 安全方向：XCTF 或 网鼎杯

**时间投入建议**：
- 大二暑假/大三上开始准备
- 集中精力打 1-2 个比赛，不要分散精力同时打多个

### 3.2 双非/二本学生

**定位**：竞赛是突围关键，必须认真对待。

**策略**：
- 竞赛是弥补学历短板的最有效手段之一。
- 必须至少有 1 个拿得出手的竞赛经历——"拿得出手"意味着国家级及以上，或者有具体的量化成绩。
- 如果没有竞赛经历，面试官的问题会集中在"你的学校我没听过"上；有了好的竞赛经历，面试官会转而问"你在竞赛中做了什么"——这是主动权的转移。

**三步走策略**：

```
第 1 步: 入门练手（1-2 周）
  目标: 快速熟悉竞赛流程和套路
  推荐: Datawhale AI 夏令营 / 和鲸社区竞赛 / LeetCode 周赛
  产出: 一段可写的项目经历

第 2 步: 正式参赛（1-3 月）
  目标: 在正规赛事中获得可量化成绩
  推荐: 天池竞赛 / 蓝桥杯 / "中国软件杯" / 各厂 Hackathon
  产出: 竞赛获奖或排名 + 技术方案文档

第 3 步: 竞赛转项目（赛后 1-2 周）
  目标: 将竞赛经历最大化复用
  操作: 
    - 整理竞赛代码为规范的 GitHub 仓库（含 README/文档/模型说明）
    - 撰写技术博客（CSDN/掘金/知乎）复盘竞赛方案
    - 提炼竞赛中的技术点写入简历项目经历模块
```

**关键心态**：即使竞赛没获奖，参与经历也完全可以写在简历上——关键是写法（见第五章）。

### 3.3 研究生

**定位**：论文 > 竞赛，但竞赛仍有价值。

**策略**：
- 如果已有高质量论文（SCI/EI/CCF），竞赛的重要性大幅降低。
- 如果论文产出不理想，竞赛可以作为项目能力的补充证明。
- 竞赛选择应与研究方向一致——做 NLP 的打 NLP 相关竞赛，做 CV 的打 CV 相关竞赛，不要把竞赛和研究做成两个不相关的方向。

### 3.4 跨专业转码

**定位**：竞赛是证明"我虽然是跨专业但我会写代码"的重要证据。

**策略**：
- 竞赛优先级：竞赛 ≈ 项目 > 学历（因为学历已经是既成事实）
- 优先参加与目标岗位直接匹配的竞赛
- 竞赛过程中的代码量和学习量本身就是转专业最有效的训练

## 四、Skill 中的竞赛工作流

当 resume-builder skill 的 Phase 1（方向确认）或 Phase 2（技能盘点）处于活跃状态时，技能应按以下流程引导用户。

### Step A：竞赛经历询问

完成方向确认和技能盘点后，should 主动询问用户的竞赛经历：

```
🎯 竞赛经历确认

你有竞赛/比赛经历吗？

A. 有 — 已获奖（请描述竞赛名称、级别、获奖情况）
B. 有 — 参加过但未获奖
C. 没有 — 但想参加
D. 没有 — 不打算参加
```

**分支逻辑**：

| 用户选择 | 后续流程 | 目标 |
|---------|---------|------|
| A | 进入 Step B — 提取竞赛内容到简历 | 将已有竞赛经历最大化利用 |
| B | 进入 Step B — 提取竞赛内容（作为项目经历处理） | 转化为项目经历 |
| C | 进入 Step C — 推荐竞赛 + 创建目标 | 帮助用户规划竞赛路径 |
| D | 跳过竞赛模块，直接进入 Phase 3 | — |

### Step A.5: WebSearch 查询竞赛含金量（选 A/B/C 均需执行）

**重要: 在推荐或提取任何竞赛之前，必须先用 WebSearch 工具查询该竞赛的最新信息。**

原因: 竞赛含金量会随时间变化（如 2025-2026 年 ACM 红利缩水、实习权重上升），不能依赖静态数据。

**查询策略:**
```
对每个待评估的竞赛，执行以下搜索:
1. "[竞赛名称] 大厂校招 简历含金量 2025 2026"
2. "[竞赛名称] [用户方向] 岗位 认可度 2025"
3. 可选: "[竞赛名称] 争议 含金量下降 水赛"（查负面评价）
```

**搜索后产出:**
```
┌──────────┬──────────┬─────────────────────┬──────────────┐
│ 竞赛      │ 含金量    │ 对 [用户方向] 岗的价值   │ 适合用户？     │
├──────────┼──────────┼─────────────────────┼──────────────┤
│ [竞赛1]   │ ★★★★    │ [基于搜索结果的评价]    │ ✅/⚠️/❌     │
│ [竞赛2]   │ ...      │ ...                 │ ...          │
└──────────┴──────────┴─────────────────────┴──────────────┘
```

**含金量判断维度:**
1. 是否被大厂面试官明确认可？（搜索结果中是否出现"直通面试""免笔试"等）
2. 是否因过度商业化/水奖泛滥导致含金量下降？
3. 对用户的具体方向是否对口？（ACM 对后端加分、对 NLP 加分有限）
4. 获奖难度 vs 时间投入比
5. 是否有"实习经历 > 竞赛经历"的趋势变化？

**记录至飞书:**
- 将搜索结论写入目标表备注（作为竞赛选择的依据）
- 方便后续会话恢复时不用重复搜索

### Step B：提取竞赛内容到简历（选 A 或 B）

**对于已获奖的竞赛（选 A）**：

提取以下信息并写入简历「竞赛/获奖」模块：

```markdown
需要向用户确认的信息:
1. 竞赛全名（不要缩写，如"ACM-ICPC 国际大学生程序设计竞赛"而非"ACM"）
2. 竞赛级别（国际/国家/省级）
3. 获奖名称和等级（金牌/银牌/铜牌/一等奖/二等奖/具体名次）
4. 你在竞赛中的角色和贡献（队长/核心编码/负责模型调优/...）
5. 使用的技术栈和方法
6. 量化结果（排名 X/总参赛队伍数 Y，成绩排名 top Z%，F1 score / Accuracy 等指标）

提取示例:
  "ACM-ICPC 亚洲区域赛（南京站）铜牌
   — 队伍排名 38/186，负责图论和动态规划题目的解题与编码"
  
  "Kaggle 'Feedback Prize - English Language Learning' 竞赛银牌
   — 排名 top 4% (86/2454)，使用 DeBERTa-v3 + 多任务学习，
     MCRMSE score 0.431，负责模型设计与集成"
```

**对于未获奖的竞赛（选 B）**：

不写入"竞赛/获奖"模块，而是从竞赛中提取技术内容，转化为项目经历写入"项目经历"模块：

```markdown
转化逻辑:
  竞赛名称 → 项目名称
  竞赛任务 → 项目目标
  技术方案 → 项目技术栈
  竞赛排名 → 项目量化结果
  竞赛代码 → GitHub 项目仓库

转化示例:
  原竞赛: 参加天池"新闻文本分类"竞赛，未获奖
  转化后项目:
    "新闻文本分类系统
     — 基于 PyTorch + Transformers 构建多类别文本分类模型，
      对比了 BERT/RoBERTa/ERNIE 三种预训练模型，
      在 THUCNews 数据集上达到 93.2% 的 Accuracy，
      使用模型蒸馏将推理速度提升 3x"
```

### Step C：推荐竞赛 + 创建目标（选 C）

当用户选择"没有竞赛经历但想参加"时，执行推荐逻辑。

**推荐参数**：

| 参数 | 来源 | 用途 |
|------|------|------|
| 目标方向 | Phase 1 确认的结果 | 确定推荐哪个方向的竞赛 |
| 学历层次 | Phase 1 确认的结果 | 确定推荐竞赛的级别 |
| 当前技能栈 | Phase 2 盘点的结果 | 匹配合适难度的竞赛 |
| 可用时间 | 需询问用户 | 确定竞赛周期 |

**推荐三段式**：

```
根据你的方向(<方向>)和学历(<学历层次>)，推荐以下竞赛路径:

📍 入门练手（1-2 周，建立竞赛经验）
  → <入门竞赛名称> — <赛道/方向描述>
  → 为什么推荐: <理由>
  → 预计投入: <时间/精力>

📍 正式参赛（1-3 月，争取获奖/排名）
  → <正式竞赛名称> — <赛道/方向描述>
  → 为什么推荐: <理由>
  → 预计投入: <时间/精力>

📍 高阶冲刺（可选，3 月+，冲高含金量）
  → <高阶竞赛名称> — <赛道/方向描述>
  → 为什么推荐: <理由>
  → 预计投入: <时间/精力>
```

**推荐示例**：

```
根据你的方向(NLP/ML)和学历(双非一本)，推荐以下竞赛路径:

📍 入门练手（1-2 周，建立竞赛经验）
  → Datawhale AI 夏令营 — NLP 赛道
  → 为什么推荐: 免费、有完整教程、有社区答疑、2 周即可完成
  → 预计投入: 每天 1-2 小时

📍 正式参赛（1-2 月，争取排名）
  → 天池"零样本分类"竞赛 — 文本分类/情感分析赛道
  → 为什么推荐: 阿里主办，认可度高，NLP 方向高度对口
  → 预计投入: 每天 2-3 小时，周末集中

📍 高阶冲刺（2-3 月，冲国际奖项）
  → Kaggle — NLP 相关竞赛（关注 feedback / text-classification 标签）
  → 为什么推荐: 含金量最高，双非简历的强力加分项
  → 预计投入: 每天 2-3 小时，需要较强的自学能力和英文阅读能力
```

**推荐后创建飞书目标**：

用户选定竞赛后，将目标写入飞书目标表。如果飞书 MCP 可用，调用 `bitable_v1_appTableRecord_create` 创建记录；如果不可用，生成本地 JSON 记录供后续同步。

### Step D：竞赛准备任务拆解

选定竞赛后，帮助用户拆解为可执行的里程碑任务。

**里程碑模板**：

```
竞赛名称: <竞赛名称>
目标: <奖项/排名目标>

里程碑:
  □ M0: 报名 + 组队 — 截止 <日期>
      - 找到 1-2 名队友（去哪儿找: 学校群/Kaggle 讨论区/Datawhale 社区/知乎私信）
      - 完成报名、阅读赛题、确认规则
      - 搭建共享代码环境（GitHub 私有仓库）
  
  □ M1: 基线复现 — 截止 <日期>
      - 跑通官方 Baseline（1-2 天）
      - 理解评估指标和数据格式
      - 建立本地训练+验证+提交的完整流程
  
  □ M2: 初步调优 — 截止 <日期>
      - 尝试 3 种以上改进方案（数据增强/模型替换/超参调优/集成）
      - 每次改进记录实验日志（模型、参数、分数变化）
      - 本地验证分数超过 Baseline 10% 以上
  
  □ M3: 深度优化 — 截止 <日期>
      - 分析 top 方案思路（查阅往届方案/论坛讨论）
      - 尝试模型集成、伪标签、对抗训练等进阶技巧
      - 本地分数接近或进入排行榜前列
  
  □ M4: 最终提交 — 截止 <比赛截止日期>
      - 选择最优 checkpoint
      - 生成最终预测文件
      - 完成提交 + 撰写技术方案文档
  
  □ M5: 赛后整理 — 截止 <比赛结束后 2 周>
      - 整理代码为规范 GitHub 仓库
      - 撰写技术博客复盘竞赛
      - 更新简历（竞赛获奖/项目经历）
```

**和 Phase 3 项目实现的关系**：

- 竞赛可以作为"项目 0"优先启动。因为竞赛有明确的截止日期和外部驱动力，比自驱项目更容易坚持。
- 竞赛周期短（1-3 月），见效快，在 Phase 3 的长期项目还在进行中时，竞赛就可以产出可写入简历的内容。
- 竞赛结束后，如果能整理出规范性好的 GitHub 仓库和技术博客，可以直接转化为一个高质量的项目经历。

**时间规划建议**：

| 阶段 | 每天投入 | 周末投入 | 主要活动 |
|------|---------|---------|---------|
| 报名+组队 | 0.5h | 1-2h | 阅读赛题、找队友、环境搭建 |
| 基线复现 | 1-2h | 3-4h | 跑通 baseline、理解数据 |
| 调优迭代 | 1-2h | 4-6h | 实验各种改进方案 |
| 冲刺提交 | 2-3h | 6-8h | 最终优化、集成、提交 |

## 五、竞赛经验 -> 简历转化规则

### 5.1 已获奖的写法

**好的写法特征**：
- 竞赛全名（含级别）+ 获奖等级 + 量化排名 + 使用的技术 + 具体指标

```
✅ 优秀示例 1（Kaggle 获奖）:
"Kaggle 'NLP with Disaster Tweets' 竞赛银牌
 — 排名 top 4% (86/2174)，使用 BERT-large + 对抗训练 + 数据增强，
   F1 score 0.847，负责模型设计与集成策略"

✅ 优秀示例 2（国内竞赛获奖）:
"天池'全球人工智能技术创新大赛'医学影像赛道 亚军
 — 团队排名 2/1687，基于 EfficientNet + 注意力机制 + 多尺度特征融合，
   最终 AUC 0.962，负责模型调优与模型集成"

✅ 优秀示例 3（ACM 获奖）:
"ACM-ICPC 亚洲区域赛（上海站）银牌
 — 队伍排名 15/210，个人在 5 小时内完成 3 道中等难度题目的解题与编码"
```

**不好的写法特征**：
- 缺少量化信息、技术细节模糊、竞赛级别不明确

```
❌ 错误示例 1:
"参加了 Kaggle 竞赛，获得银牌"
→ 问题: 哪个竞赛？排名多少？用了什么技术？完全没信息量。

❌ 错误示例 2:
"蓝桥杯获奖"
→ 问题: 什么组别？什么级别（国赛/省赛）？几等奖？太模糊。

❌ 错误示例 3:
"参加过多项编程竞赛并取得优异成绩"
→ 问题: 这是废话，"优异"没有定义，不如不写。
```

### 5.2 未获奖的写法

没获奖也能写，关键是**把竞赛转化为项目经历**，而不是写成"参加过但没获奖"。

```
✅ 正确转化:
"文本情感分析系统
 — 基于 PyTorch + Transformers 构建多类别情感分析模型，
  在 AI Challenger 情感分析数据集上对比了 BERT/RoBERTa/ERNIE 三种预训练模型的效果，
  最终方案 F1 score 达到 0.873（对比 baseline 提升 12%），
  推理延迟优化至 15ms/条"

❌ 错误写法:
"参加了 AI Challenger 情感分析竞赛，未获奖"
→ 问题: 自曝其短，完全没有必要强调"未获奖"。
```

### 5.3 简历中竞赛模块的位置

简历中竞赛模块的位置取决于其含金量：

| 竞赛含金量 | 位置 | 说明 |
|-----------|------|------|
| 高（国际级获奖/Kaggle 银牌+） | 教育经历之后、实习经历之前 | 作为核心亮点前置 |
| 中（国家级获奖） | 实习经历之后、项目经历之前 | 标准位置 |
| 低（省级获奖/参与） | 项目经历之后 | 作为补充信息 |

### 5.4 竞赛经历的条数

- **1-2 条高质量** >> 5 条低质量。简历上最多 3 条，超过会挤占项目和实习的空间。有多条时只保留级别最高/成绩最好的。

## 七、常见问题与误区

### Q1: 竞赛和项目哪个更重要？

**答**：竞赛获奖的简历价值高于同等投入的一般项目。但有机会成本——2 个月打竞赛如果没获奖，产出不及 2 个月做高质量项目。学习能力强、愿接受挑战则优先竞赛；时间紧张、需确定性产出则优先项目。

### Q2: 没有队友怎么办？

**答**：天池、Kaggle 支持个人参赛。找队友渠道：学校 ACM 实验室/社团、Datawhale 社区、Kaggle 讨论区、知乎/小红书搜"[竞赛名] 组队"、赛题官方 QQ 群。找不到就个人参赛。

### Q3: 竞赛需要花钱吗？

**答**：大部分数据竞赛（Kaggle/天池/和鲸）免费。蓝桥杯等需报名费（300-500 元，学校可能报销）。Kaggle 可能需要 GPU 算力（Colab 免费版或 AutoDL 租用，几十元够用）。ACM/ICPC 通常学校包报名费和差旅费。

### Q4: 竞赛和上课冲突怎么办？

**答**：竞赛对求职的帮助远大于大部分课程成绩（除非保研）。对于明确要就业的同学，适当减少课程投入是合理策略。底线：不挂科。

### Q5: 大一/大二可以参加竞赛吗？

**答**：大一建议从蓝桥杯入门组、LeetCode 周赛、Datawhale 夏令营开始。大二可正式参加天池/Kaggle/ACM 区域赛等正式赛事。大二暑假是竞赛的黄金时间窗口。

### Q6: 秋招前多久参赛最合适？

**答**：最佳时间是秋招前 3-6 个月完成竞赛（大三下 3-6 月）。秋招临近（< 2 个月）不建议新开竞赛，应专注优化简历和刷题。春招前 2-4 个月（大四上 11-12 月）也是好时机。
