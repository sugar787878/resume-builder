# 职业方向探索指南

> 面向应届生的求职方向决策树、简历骨架模板与关键词库。
> 本文件为 resume-builder skill 的核心参考文件。

---

## 一、方向探索决策树

使用方式：从大类出发，逐层回答选择题，抵达具体岗位标签。选"不确定"时回溯到上一层换分支继续探索。

### 0. 学历分层（所有方向通用）

在进入具体方向决策树之前，首先确认学历层次，这会影响后续所有决策（简历板块顺序、项目数量、实习策略等）。

```
学历层次
├─ 985 / 双一流
│  └─ 策略: 教育前置，项目重质量(2-3个)，大厂实习优先
│
├─ 211
│  └─ 策略: 教育前置，项目+实习并重(3-4个项目)
│
├─ 双非一本
│  └─ 策略: 教育后置，项目驱动(3-4个项目+1段实习最低)，
│           技术栈聚焦1个方向，参考 resume-strategies.md
│
└─ 二本/专科
   └─ 策略: 教育后置/弱化，项目驱动(4-5个项目+1-2段实习)，
            必须有明星项目，考虑外包/子公司/内推渠道
```

> 详细策略参见 `references/resume-strategies.md`。

### 1. 后端开发

```
后端开发
├─ 语言/技术栈偏好？
│  ├─ Java/Spring → 国内大厂最主流，岗位最多
│  ├─ Go → 云原生/基础设施/字节腾讯系
│  ├─ C++ → 交易系统/游戏后端/基础架构
│  ├─ Python → 工具平台/数据处理/初创后端
│  └─ 不确定 → 推荐 Java（覆盖面最广）
│
├─ 业务领域偏好？
│  ├─ 电商/交易 → 高并发/分布式事务/库存系统
│  ├─ 金融/支付 → 强一致性/风控/清算
│  ├─ 社交/内容 → 高吞吐/Feed流/推荐引擎对接
│  ├─ 企业服务(SaaS/B2B) → 多租户/权限/RBAC
│  ├─ 云基础设施 → 存储/计算/网络/中间件
│  └─ 不确定 → 电商领域通用性强
│
├─ 公司规模倾向？
│  ├─ 大厂(阿里/腾讯/字节/美团/快手/百度) → 流程规范/细分深/资源足
│  ├─ 中厂(小红书/B站/滴滴/京东/拼多多/携程) → 成长空间大/接触面广
│  ├─ 创业公司/AIGC新锐(Kimi/DeepSeek/Minimax/智谱) → 快速成长/高风险高回报
│  └─ 外企中国分部(Google/Microsoft/Amazon/Stripe/Shopee) → WLB/全球化视野
│
└─ 岗位标签生成: [语言] 后端开发 - [领域]方向
   例: Java后端开发 - 电商交易方向 (中厂)
```

### 4. 运维/DevOps/SRE

```
运维/DevOps/SRE
├─ 平台与技术栈？
│  ├─ Linux + 云原生(K8s/Docker) → 最普及
│  ├─ Windows + Azure → 外企/游戏行业
│  └─ 不确定 → Linux + 云原生

│  ├─ SRE(站点可靠性) → 稳定性/监控/容量规划/故障处理
│  │  └─ 关键词: SLI/SLO/On-call/故障演练/混沌工程

│  │  └─ 关键词: GitLab CI/GitHub Actions/Jenkins/ArgoCD
│  ├─ 云平台/基础设施 → IAC(Terraform)/多云/FinOps
│  │  └─ 关键词: Terraform/Ansible/多云管理/成本优化
│  ├─ 安全运维(DevSecOps) → 安全扫描/合规/密钥管理
│  └─ 不确定 → DevOps覆盖面广/SRE待遇较高
│
├─ 业务规模？
│  ├─ 大规模(百万DAU+) → 高可用架构/多活/容灾
│  ├─ 中等规模 → 自动化建设/效能提升
│  ├─ 初创公司 → 全栈运维/从0到1
│  └─ 不确定 → 先在大厂积累大规模经验
│
└─ 岗位标签生成: [侧重点]工程师 - [平台]
   例: SRE工程师 - 云原生 (大规模业务)
```

## 四、通用简历原则(不区分方向)

### DO(要做)
- 用**STAR法则**(情境-任务-行动-结果)写每段经历
- **量化一切**: QPS提高了X%、成本降低了Y%、覆盖了Z万用户
- 简历**一页原则**(应届生)，最多不超过两页
- 技术栈用**精通/熟悉/了解**分三档，不要全部写"精通"
- 针对不同岗位**定制不同版本**的简历
- 附上**可点击的链接**: GitHub、作品集、博客、项目Demo

### DON'T(不要做)
- 不要放非职业照片
- 不要写"吃苦耐劳/性格开朗/沟通能力强"等软技能套话
- 不要用进度条/百分比表示技能掌握程度(视觉污染)
- 不要列举大学所有课程(只列相关性高的3-5门)
- 不要写空洞的自我评价(除非有具体成就支撑)
- 不要在同一份简历投所有岗位
